## Today's Work

### 1. Design: Show Everything, Don't Make People Choose

Most case converters give you a dropdown: pick a style, get one result. I went the other
way ‚Äî one input, and **all nine conversions rendered at once**, each with its own copy
button. When you're converting a variable name you often don't know which style you want
until you see them side by side, and it removes a click from every use. It's also less code:
no mode state to manage.

### 2. The Hard Part: Tokenizing Input That's Already Styled

The naive implementation splits on spaces. That fails immediately, because the most common
real input to this tool isn't prose ‚Äî it's an identifier that's *already* in some style.
Someone pastes `helloWorldFoo` and wants `hello_world_foo`. Splitting on spaces yields one
token and produces garbage.

So the core of the tool is a tokenizer that understands existing styles:

- Insert a break between a lowercase/digit and an uppercase letter: `fooBar` ‚Üí `foo Bar`.
- Handle acronym boundaries: `HTTPServer` ‚Üí `HTTP Server` (without this, `HTTPServerError`
  becomes one meaningless token).
- Treat any non-letter/non-digit as a separator, which covers `_`, `-`, spaces and
  punctuation in one rule.

Using Unicode property escapes (`\p{Ll}`, `\p{Lu}`, `\p{L}`, `\p{N}`) rather than ASCII
ranges ‚Äî the same lesson as yesterday's counter: the site is bilingual, so the regexes
shouldn't assume English.

### 3. A Bug My Own Tests Caught

I tested the conversions in Node before wiring up any UI, and the programmer styles all
round-tripped correctly. But the prose styles were wrong: Title Case of `helloWorldFoo`
came out as `Helloworldfoo`.

The cause was a real design tension. Title Case and Sentence case must operate on the
**original text** to preserve punctuation and spacing (`hello there. how are you?` ‚Üí
`Hello There. How Are You?`), while the programming styles operate on **tokens**. My first
version simply lowercased the original text, so camelCase humps disappeared.

The fix was to split the difference: break camel humps into spaces *first*, then apply the
prose transform to that. Now `helloWorldFoo` ‚Üí `Hello World Foo` **and** prose keeps its
punctuation. Re-tested and also confirmed an apostrophe edge case (`o‚Äôclock` doesn't get
capitalized mid-word).
