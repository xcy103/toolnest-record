## Today's Work

### 1. Secure Randomness (the part that matters)

A password generator lives or dies on its randomness. `Math.random()` is the wrong tool —
it's not cryptographically secure and its output can be predicted. The right one is
**`crypto.getRandomValues()`**, the same Web Crypto API the hash tool used, which fills a
typed array with cryptographically strong random bytes.

The subtle bug I made sure to avoid is **modulo bias**. The naive `random % poolSize`
skews toward the lower indices whenever `poolSize` doesn't evenly divide 2³², so some
characters become slightly more likely than others. The fix is **rejection sampling**:
compute the largest whole multiple of `poolSize` that fits in the 32-bit range, and throw
away (re-roll) any value at or above it, so every character is exactly equally likely. I
verified this with a quick distribution test — 60k single-digit draws came out within ~4%
of uniform, versus the visible skew a modulo would give.

### 2. The Tool

Built `/password` with:

- A **length slider** (4–64) and **character-set toggles** — lowercase, uppercase,
  numbers, symbols — plus an "avoid ambiguous characters" option that strips `I l 1 O 0`
  for passwords you might have to read or type by hand.
- A **strength meter** derived from Shannon entropy (length × log₂ of the pool size),
  bucketed into weak / fair / good / strong. It reflects the actual math, not a vague guess.
- **Regenerate** and **copy** buttons, and a clear error when every character type is
  switched off (nothing to generate from).

Like the timezone tool, generation runs **client-side only** (in an effect), never during
render — random output on the server wouldn't match the client and would cause a hydration
mismatch. Hit the familiar "no setState directly in an effect" lint rule again and applied
the now-standard fix: a named function inside the effect.

`en.json` and `zh.json` as I went. Nice confirmation that the earlier work paid off:
registering the tool made it appear on the home page, in its category, **and in the
sitemap** automatically, in both languages, with no extra wiring.
