## Today's Work

### 1. Learning JSON Parsing & Serialization (morning)

Started with the two built-in halves of JSON handling in JavaScript:

- **`JSON.parse(text)`** takes a JSON string and turns it into an in-memory value (object, array, string, number, …), throwing a `SyntaxError` if the text isn't valid JSON.
- **`JSON.stringify(value, replacer, space)`** goes the other way. The third argument, `space`, is the key to formatting: pass a number (like `2`) and it pretty-prints with that many spaces of indentation per level; omit it and you get the compact, minified form with no whitespace.

So "format" and "minify" are really the same round-trip — parse to a value, then stringify — differing only in whether you pass the `space` argument. Validation comes for free: if `JSON.parse` doesn't throw, the input is valid JSON, full stop.

### 2. Making Parse Errors Useful (morning)

The interesting part was error handling. A raw `SyntaxError` message like *"Unexpected token } in JSON at position 42"* is intimidating and the "position" is a raw character offset, which is useless to a human staring at a multi-line document. I learned I can catch the error, pull the `position N` number out of the message, and convert that offset into a **line and column** by counting newlines in the text before that point. That turns a cryptic message into something like "syntax error near line 3, column 7" — a real, actionable hint. Worth noting this string-parsing of the message is engine-specific (V8/Chrome phrasing), so I kept a plain fallback message for when the pattern doesn't match.

### 3. Building the JSON Tool (evening)

Built the tool page (`/json`) using the concepts from the morning:

- Format / minify mode switch, both built on the same parse-then-stringify round-trip.
- An indent selector (2 or 4 spaces) that only appears in format mode, since it's meaningless when minifying.
- Live validation: as you type, valid JSON is formatted/minified instantly, and invalid JSON shows the friendly line/column error message instead of crashing or showing garbage.
- A copy button on the output and a clear button on the input, and a bigger input area (8 rows) since JSON documents tend to be larger than a line of text.

This again reused all the shared components (`ToolLayout`, `ToolPanel`, `CopyButton`) from Day 6 with zero changes — three tools in now, and the scaffolding is still carrying its weight.
