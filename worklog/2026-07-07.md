## Today's Work

### 1. Learning URL Encoding (morning)

Started by learning what URL encoding — also called **percent-encoding** — actually is and why it exists. The core idea: a URL is only allowed to contain a limited set of ASCII characters. Anything outside that set (spaces, Chinese characters, and reserved delimiters like `/ ? & = #` when they appear in data rather than as structure) has to be escaped as `%` followed by the two hex digits of each UTF-8 byte. That's why a space becomes `%20` and a Chinese character turns into several `%XX` groups — each byte of its UTF-8 encoding.

An important distinction I made sure to understand: **reserved characters vs. the data inside them**. Characters like `/`, `?`, `&`, `=`, `#` have structural meaning in a URL. If they're part of a value (say a query-parameter value that itself contains an `&`), they must be encoded so they aren't mistaken for structure; if they're the URL's actual skeleton, they must be left alone.

### 2. Two Ways to Encode in the Front-end (morning)

Looked into how to do this in the browser and found there are two built-in pairs, which trip people up:

- **`encodeURIComponent` / `decodeURIComponent`** — escapes almost everything, including the reserved delimiters `/ ? & = #`. This is the right choice for a single **component**: one query-parameter value, one path segment.
- **`encodeURI` / `decodeURI`** — deliberately leaves the reserved structural characters alone, so it's for encoding a **whole URL** that already has its skeleton in place (e.g. a link with Chinese or spaces in it).

Picking the wrong one is the classic bug: `encodeURI` on a parameter value leaves a stray `&` that breaks the query string, while `encodeURIComponent` on a full URL mangles the `://` and slashes. I also learned that `decodeURIComponent` throws on malformed input (a `%` not followed by two valid hex digits), which — like yesterday's `TextDecoder` trick — is a clean way to validate pasted input instead of silently returning garbage.

### 3. Building the URL Tool (evening)

Built the tool page (`/url`) using the concepts from the morning:

- Encode / decode mode switch, with the current result carried over when switching so it's easy to round-trip.
- A second **scope** switch — "component / parameter value" vs. "whole URL" — mapping to the `encodeURIComponent` and `encodeURI` pairs, with a one-line hint explaining when to use each. This makes the two-functions distinction a first-class part of the UI instead of a hidden gotcha.
- Full UTF-8 support (Chinese, spaces, emoji) since percent-encoding works on the underlying bytes.
- Input validation with friendly error messages — a malformed percent-escape shows a clear prompt instead of crashing.
- A copy button on the output and a clear button on the input.
