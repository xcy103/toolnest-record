## Overview

Third tool of Wave 1: a word and character counter. It also opens a new category —

## Today's Work

### 1. A New Category, Cleanly

Before writing the tool I checked what adding a category actually costs now. It turned out
to be exactly what I'd hoped when I refactored the registry during the i18n work: add
`"text"` to the `CategoryKey` union, add one `categories.text` entry to each message
catalogue (Text / 文本), and tag the tool with `categoryKey: "text"`. The home page groups
by category automatically, so the new section appeared with no layout changes. That's the
payoff of keeping the registry as pure structure — growth is additive, not surgery.

### 2. The Real Problem: Counting Words in Two Languages

The counter itself is simple arithmetic except for one genuinely interesting part: **what is
a "word"?** The naive answer — split on spaces — is an English assumption. Chinese doesn't
put spaces between words; "你好世界" is four characters with no delimiters, and splitting on
whitespace would count that whole run as a single "word", which is wrong.

Since the site is bilingual, the counter has to be too. I learned that the robust approach
is script-aware counting:

- Count **Han (CJK) characters individually** — for Chinese, character count is the
  meaningful "word" measure.
- For everything else, count **runs of letters or digits** as one word each, using Unicode
  property escapes (`\p{Script=Han}`, `\p{L}`, `\p{N}`) rather than ASCII-only `\w`, so
  accented Latin, Cyrillic, etc. are handled too.
- Add the two together for mixed text.

I verified this against a handful of cases before trusting it: `"Hello world, this is a
test."` → 6, `"你好世界这是测试"` → 8, mixed `"Hello 世界 foo"` → 4, and 200 repeated words
→ 200 (and 1 minute reading time). The mixed case is the one a lazy implementation gets
wrong, so it was the important one to check.

### 3. Building the Tool

Built `/word-counter`:

- A stats grid (six tiles): characters, characters without spaces, words, lines,
  paragraphs, and estimated reading time (≈200 words/minute).
- A large text area below; stats update live via a `useMemo` over the input.
- Paragraph counting splits on blank lines and ignores empty runs.

No randomness here, so unlike the last two tools there's no client-only dance — the stats
are a pure function of the input, so server and client agree and `useMemo` is enough.

### 4. Verification

Unit-checked the counting function on English, Chinese, mixed, empty, and multi-paragraph
inputs (all correct), then lint + build (27 static pages now). Served the build and checked
the visible HTML in both locales: the tool renders with translated tiles, and the new
**Text / 文本** category shows up on both home pages. Committed and pushed to `toolnest`.
