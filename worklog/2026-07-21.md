## Today's Work

### 1. Learning: What a UUID Actually Is

Took the morning to understand what I was generating rather than treating `randomUUID()` as
a black box. A UUID is a 128-bit identifier written as 32 hex digits in five hyphenated
groups (`8-4-4-4-12`). The common flavour — and the only one the browser produces natively —
is **version 4**, which is almost entirely random: 122 random bits, with a few bits fixed to
mark the version (`4`) and variant. With that much entropy, two randomly generated v4 UUIDs
colliding is effectively impossible, which is why they're used as database keys, request
IDs, idempotency keys and the like without a central authority handing them out.

The browser gives me `crypto.randomUUID()`, which uses the same cryptographic RNG as
yesterday's password tool — so the values are properly random, not `Math.random()` quality.

### 2. Building the Tool

Built `/uuid`:

- A count slider (1–50) for generating one or many at once — bulk generation is the common
  real need when seeding data.
- Two format toggles: **uppercase** and **remove hyphens**, the two variations people
  actually paste into different systems.
- A regenerate button and a copy button that grabs the whole list (newline-separated).

The output textarea grows with the count (clamped between 3 and 12 rows) so a single UUID
doesn't sit in a huge box and fifty don't overflow.

### 3. The Recurring Client-Only Lesson

Same trap as the password generator, and I'm starting to internalise it as a rule: **random
output can't be produced during render.** If the server generated UUIDs and the browser
generated different ones on hydration, React would complain about the mismatch. So the list
starts empty and is filled by an effect that only runs on the client. I verified this held:
the served static HTML contains *no* UUID (checked with a regex for the v4 pattern) — the
values only appear after the page hydrates. Applied the project's usual shape too, calling
setState from a named function inside the effect to satisfy the lint rule.
