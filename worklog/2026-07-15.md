# Work Log · July 15, 2026 (Wednesday)

**Hours:** ~4

## Overview

Day 2 of the i18n work. Yesterday laid the routing plumbing with no visible change; today
the site actually became bilingual for the first time — the home page, navigation and
footer now render in real English on `/en` and stay Chinese on `/zh`.

## Today's Work

### 1. A Correction to the Plan

The roadmap said "Day 3: create `messages/en.json`; Day 4: translate `zh.json`". Once I
started, that split didn't survive contact with reality:

- The Chinese copy **already exists** in the code, so putting it in `zh.json` isn't
  translation work — it's copy-paste. The actual translating is Chinese → English, which
  is today's job either way.
- Worse, shipping only `en.json` would leave `zh.json` empty, and `/zh` would blow up with
  missing-message errors. The plan would have ended the day with a broken locale.

So I re-cut the work by **scope instead of by language**: today covers the site chrome,
home page and tool registry — *both* languages, fully working. The seven tool pages come
tomorrow. Each step leaves the site shippable, and it lets me prove the pattern on a small
surface before repeating it seven times.

### 2. Designing the Message Catalogue

Set up `messages/en.json` and `messages/zh.json` with a shared key structure: `site`,
`nav`, `home`, `toolCard`, `toolLayout`, `footer`, `categories`, and `tools.<slug>`.

The interesting problem was the home page counter, which isn't a plain string — it has
**bold numbers embedded in a sentence** ("7 tools planned, 7 live and growing"). Word order
differs between English and Chinese, so I can't concatenate fragments. Learned that
next-intl handles this with **rich text**: the message holds tags like
`<strong>{total}</strong>`, and `t.rich(...)` maps each tag to a React element. The
translator controls the sentence; the code controls the styling.

I also hit a real bug while writing it: I'd named a tag `<live>` while also passing a value
called `live` — the same name can't be both a placeholder and a tag. Renamed the tag to
`<em>`.

### 3. Refactoring the Registry

The biggest structural change was `lib/tools.ts`. It used to hold each tool's Chinese name,
description and category as literal strings. Now the registry carries only **structure** —
slug, href, icon, category key, availability — and the display strings live in the message
catalogues under `tools.<slug>`. This felt like the right separation: the registry is data
about *what exists*, not about *what it's called in a given language*.

### 4. Wiring Up the Components

Converted Navbar, Footer, ToolCard, ToolLayout, ThemeToggle (its accessibility labels were
hardcoded Chinese too) and the home page to read from translations. A useful thing I
confirmed by reading next-intl's source: when `NextIntlClientProvider` is rendered from a
Server Component it **automatically loads the messages**, so client components like the
theme toggle just work without me threading anything through.

I also localized the page **metadata** with `generateMetadata` — so `<title>` and the meta
description now follow the locale. That was scheduled for Day 5, but doing it now actually
*removed* duplication rather than adding work, since the tagline no longer had to live in
two places.

### 5. Verification

Built and served for real, then checked both locales:

- `/en` → `<title>ToolNest · Free Online Tools`, "All Tools", "Free to use", "Encoding",
  "Base64 Encode / Decode", English meta description ✅
- `/zh` → `<title>ToolNest · 免费在线工具箱`, 全部工具, 免费使用, 编码, Base64 编解码,
  Chinese meta description ✅

Then grepped the chrome for any leftover Chinese characters to be sure nothing was missed —
only the tool pages remain, which is tomorrow's scope by design.

## Outcome

ToolNest is genuinely bilingual above the fold: home page, nav, footer and metadata all
follow the URL's locale. The lesson worth keeping is the plan correction — the roadmap's
day split was written before I understood the material, and following it literally would
have shipped a broken `/zh`. Re-cutting by scope kept every step working.
