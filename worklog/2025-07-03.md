# Work Log · July 3, 2025

**Hours:** ~4

## Overview

Day 5 of ToolNest, and the start of Phase 1. The day split into two halves: a morning of studying the front-end framework I'm building on, and an evening of hands-on development. The goal for the build was to give the site a real front door — a proper home page and a shared layout that every future tool will plug into — so I deliberately kept the scope to the site skeleton, no individual tool yet, to get the foundation right before tools start landing one by one.

## Today's Work

### 1. Front-end Framework Study (morning)

Spent the morning learning the framework behind ToolNest (Next.js with the App Router). Focused on the pieces I'd need right away: how routing works from the file structure (each folder under `app/` becoming a page), how a shared root layout wraps every page so common UI like the nav bar and footer only has to be written once, how the page is broken into reusable components, and the split between server-rendered and interactive client components. Also got a clearer picture of how styling works with Tailwind CSS and how the whole thing builds into a fast static site for Vercel. Taking the time to understand these concepts up front — rather than copy-pasting — made the afternoon's build go much more smoothly.

### 2. Shared Layout & Navigation (evening)

Built the site-wide layout that wraps every page: a top navigation bar (ToolNest logo, a link to all tools, a GitHub link) and a footer with the company name and "front-end only · data never leaves your device". Putting this into shared components now means every future tool page looks consistent for free instead of repeating markup.

### 3. Dark Mode

Added a one-click light/dark theme toggle. The choice is remembered in the browser and defaults to the system preference, with a small script that applies the theme before the page renders so there's no flash of the wrong colors on load.

### 4. Home Page & Tool Directory

Built the card-style home page: a hero section introducing ToolNest, and a responsive grid of tool cards grouped by category (one column on phones, up to three on desktop). To keep the site honest as it grows, I set up a central tool registry — every planned tool is registered in one place and the home page renders from it automatically. Tools that aren't built yet show as a dimmed "coming soon" card; as each tool ships, flipping one flag makes its card go live, so the home page will visibly fill in day by day, matching the roadmap's daily rhythm.

### 5. Version Control & Push

Continued the staged-commit workflow — separate conventional commits for the tool registry, the shared layout, and the home page — then pushed everything to the `toolnest` repo, so the live site and the commit history both moved forward together.

## Outcome

ToolNest now has a real home page and a reusable layout foundation, and I understand the framework well enough to keep building on it independently. The structure is in place for tools to be added quickly and consistently from here on.
